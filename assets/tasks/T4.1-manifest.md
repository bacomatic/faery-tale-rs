# T4.1 — Manifest + driver

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
All extractor tasks (Wave 1 + Wave 2).

## Inputs (read-only)
- Every extractor under `tools/`
- The emitted `assets/` tree

## Implementation
Create `tools/build_assets.py`:
1. Run every extractor in dependency order.
2. Write `assets/manifest.json`: per emitted file — path, type, byte size, **SHA-256**, source reference.
3. Output must be **deterministic** (stable bytes across re-runs).

## Outputs
- `tools/build_assets.py`, `assets/manifest.json`
- `tools/tests/test_build_assets.py`

## Implementer self-check
- Manifest covers every emitted file; re-run yields identical checksums.

## Verification (DIFFERENT agent)
1. **Determinism:** run `build_assets.py` twice from clean; diff `manifest.json` — byte-identical.
2. **Coverage:** walk `assets/` yourself and assert every file (except docs you decide to exclude)
   appears in the manifest; assert no manifest entry points to a missing file.
3. **Checksum integrity:** independently SHA-256 a random sample of 5 files and compare to the manifest.
4. Confirm each entry has a non-empty source reference.
5. Re-run pytest clean.
Report PASS/FAIL with the determinism diff and the sampled checksums.
