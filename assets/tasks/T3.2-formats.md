# T3.2 — Format spec

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
All of Wave 1 + Wave 2 (this documents their outputs). Best done after T3.1 too.

## Inputs (read-only)
- The emitted `assets/` resources, `assets/shaders/`, `experiment/shaders/FINDINGS.md`
- `tools/check_md_links.py` (link checker)

## Implementation
Author `assets/FORMATS.md` + `assets/formats/*.md` — one section per resource type:
- field meanings, units, coordinate systems;
- transparency convention (**index 31**);
- **palette-effects matrix**: which effects are shader-doable on RGBA and how — night vegetation
  boost on indices 16–24 needs the **1-bit `highlight_mask`** (not the indexed path);
  cross-link `assets/shaders/` and `experiment/shaders/FINDINGS.md`;
- audio synth model (period table, VBL tempo, envelopes);
- `highlight_mask` format (1 bit/px, index ∈ 16–24, transparency follows 31) under `formats/palettes.md`.

## Outputs
- `assets/FORMATS.md`, `assets/formats/*.md`

## Implementer self-check
- Every shipped resource type has a spec section; `check_md_links.py` passes.

## Verification (DIFFERENT agent)
1. Enumerate every resource type actually present under `assets/` and confirm each has a
   `formats/*.md` section — flag any undocumented type.
2. Run `tools/check_md_links.py` and confirm zero broken links (incl. cross-links to shaders/experiment).
3. **Spec-vs-reality spot-check:** pick 3 sections (e.g. sprites atlas, palette entry, music event)
   and confirm the documented field names/units exactly match a real emitted JSON file.
4. Confirm the `highlight_mask` and palette-effects matrix are documented and consistent with T3.1.
Report PASS/FAIL with the undocumented-type list (if any).
