# T0.2 — C-array extraction baseline

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
None.

## Inputs (read-only)
- `tools/extract_table.py` (existing generic C-array extractor)
- `src/fmain.c`, `src/fmain2.c`, `src/ftale.h`

## Implementation
1. Confirm/extend `extract_table.py` so it can pull any named C array as raw nested ints
   (no field semantics yet) — handle 1-D and N-D arrays, hex/decimal/char literals.
2. Smoke-extract `diroffs` (16 entries) to a fixture JSON to validate the parser.

## Outputs
- Verified/extended `tools/extract_table.py`; a `diroffs` fixture dump; `tools/tests/` parse test.

## Implementer self-check
- `diroffs` parses to the correct shape and values; pytest covers the parse.

## Verification (DIFFERENT agent)
1. Open `src/fmain.c`/`fmain2.c`, locate `diroffs` by hand, and manually transcribe the
   first and last few entries.
2. Diff your hand transcription against the Implementer's fixture JSON — must match exactly.
3. Pick a **second** array the Implementer did NOT test (e.g. `fallstates`) and extract it
   yourself with the tool; confirm shape/count plausible vs source.
4. Re-run the parse pytest from clean.
Report PASS/FAIL with the entries you transcribed.
