# T2.6 — Music + instruments extractor

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
T0.1.

## Inputs (read-only)
- `game/songs` (5,984 B): 28 packed track event streams
- `game/v6` (4,628 B): 8×128B waveforms + 10×256B envelopes
- `songs.rs` / `audio.rs` (Rust oracle)

## Implementation
Create `tools/extract_music.py`:
1. `game/songs`: 28 tracks, each `i32 packlen` (**big-endian**) + `packlen×2` bytes of
   `(command, value)` events → JSON (note / rest / set-instrument / tempo / end + loop flag).
   Include the Paula period-table reference.
2. `game/v6`: bytes 0–1023 = 8×128 **signed** waveform samples → `waveforms.json`;
   bytes 1024–3583 = 10×256 envelope tables → `envelopes.json`.

## Outputs
- `assets/audio/music/<track>.json` ×28
- `assets/audio/instruments/{waveforms,envelopes}.json`
- `tools/tests/test_extract_music.py`

## Implementer self-check
- BE parse correct; 8 waveforms / 10 envelopes; diff vs `songs.rs`; pytest green.

## Verification (DIFFERENT agent)
1. **BE sanity:** read the first track's `packlen` bytes yourself, assert big-endian decode and
   that the byte count consumed == `4 + packlen*2`; confirm 28 tracks total tile the file.
2. **Rust oracle:** dump `songs.rs` events for 2 tracks, diff vs JSON.
3. **Instruments:** assert exactly 8 waveforms (128 signed each) and 10 envelopes (256 each);
   confirm signedness on a sample with a value > 127 in source.
4. Re-run pytest clean.
Report PASS/FAIL with the tracks diffed.
