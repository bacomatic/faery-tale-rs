# T2.5 — World data extension

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
T0.1.

## Inputs (read-only)
- `tools/decode_map_data.py` (existing, ~1,467 lines of map/sector/terra logic)
- ADF `game/image` (sector/map/terra blocks)
- `world_data.rs` (Rust oracle)

## Implementation
Extend `tools/decode_map_data.py` to emit per-region JSON:
1. Sector tile-index grid.
2. Region map.
3. Terra/collision flags: **high nibble = feature type, low nibble = mask-application mode.**

## Outputs
- `assets/world/*.json` (per region)
- `tools/tests/test_decode_map_data.py`

## Implementer self-check
- Terra decode matches `world_data.rs` on spot regions; pytest green.

## Verification (DIFFERENT agent)
1. **Rust oracle:** for one outdoor region (sector=32) and one indoor (sector=96), dump
   `world_data.rs` terra/sector output and diff vs JSON.
2. Confirm nibble split: pick 3 terra entries, hand-decode high/low nibbles, compare to JSON fields.
3. Sanity: sector grid dimensions and region count match the `file_index`/memory notes.
4. Re-run pytest clean.
Report PASS/FAIL with the regions diffed.
