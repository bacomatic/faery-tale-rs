# T4.2 — Verification harness (end-to-end)

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
T4.1 (manifest/driver).

## Inputs (read-only)
- All extractors + `tools/build_assets.py`
- Rust oracles: `tile_atlas.rs`, `iff_image.rs`, `palette.rs`, `songs.rs`, `audio.rs`, `font.rs`
- `experiment/shaders/compare.py`, existing `sprite_output/`

## Implementation
Wire up the plan's Verification section as repeatable scripts/tests under `tools/tests/`:
1. **Round-trip:** indexed PNG → indices == raw bitplane decode; RGBA == palette-applied indices.
2. **Rust oracle diffs:** Python export vs each Rust decoder.
3. **Sprite regression:** new sprite PNGs vs `sprite_output/` (730 files).
4. **End-to-end:** `tools/run.sh build_assets.py --game-dir ../faery-tale-rs/game` regenerates
   the full bundle clean; `manifest.json` validates.

## Outputs
- Verification scripts/tests under `tools/tests/`; an end-to-end runner.

## Implementer self-check
- Full bundle regenerates clean; all oracle diffs zero; manifest validates.

## Verification (DIFFERENT agent)
1. From a **clean checkout/clean `assets/`**, run the end-to-end command yourself; confirm the
   bundle regenerates with no errors and identical manifest checksums.
2. Confirm each oracle diff in the harness genuinely fails when fed deliberately corrupted input
   (i.e. the checks have teeth — not no-ops).
3. Spot-open a tile atlas, a sprite sheet, a screen; play one SFX WAV.
4. Confirm the sprite-regression check actually compares against `sprite_output/` and reports zero diff.
Report PASS/FAIL, including the negative-control result from step 2.
