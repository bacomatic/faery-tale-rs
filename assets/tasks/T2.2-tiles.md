# T2.2 — Background tile atlas extractor

> Read [`_SHARED.md`](_SHARED.md) first. Two-agent task: Implementer + separate Verifier.

## Dependencies
T0.1.

## Inputs (read-only)
- ADF `game/image` (image_mem groups)
- `file_index[]` / `faery.toml [[world.region]]` (region → image-group block numbers)
- `tile_atlas.rs` (Rust oracle), `experiment/shaders/` (16×16 verification)

## Implementation
Create `tools/extract_tiles.py`. Per region:
1. 256 tiles, 16×16, **5 bitplanes**, group-major then plane-major.
2. Offset formula: `offset(T,P,R) = (T/64)*20480 + P*4096 + (T%64)*64 + R*2`.
3. Emit indexed PNG + RGBA PNG + 1-bit highlight mask (index ∈ 16–24) + `tiles.json`.

## Outputs
- `assets/tiles/region_<NN>/{atlas_indexed.png, atlas_rgba.png, atlas_highlightmask.png, tiles.json}`
- `tools/tests/test_extract_tiles.py`

## Implementer self-check
- Indexed decode matches `tile_atlas.rs`; RGBA == palette-applied; mask correct; pytest green.

## Verification (DIFFERENT agent)
1. **Rust oracle:** dump `tile_atlas.rs` atlas for one region (small harness/existing test) and
   pixel-diff against `atlas_indexed.png` — zero diff.
2. **RGBA consistency:** assert `atlas_rgba.png` == region palette applied to `atlas_indexed.png`
   (use the correct color-31 override for region 4 / region 9).
3. **Mask:** assert mask bits set exactly where index ∈ 16–24.
4. **Offset spot-check:** hand-compute the byte offset for tile 65, plane 2, row 3 and confirm
   the Implementer reads the same byte.
5. Re-run pytest clean.
Report PASS/FAIL with the region checked and the diff result.
