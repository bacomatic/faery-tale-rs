## Research Topics

### 1. Main Game Loop & Tick Structure
- **Files**: fmain.c
- **Data structures**: struct shape, struct fpage
- **Key functions**: main (fmain.c:1129)
- **Scope**: large (the `main()` function is ~1500 lines spanning fmain.c:1129-2623)
- **Dependencies**: touches nearly all other topics
- **Notes**: The main loop at fmain.c:1129 is monolithic — handles input, movement, rendering, combat, AI, encounters, door transitions, day/night, hunger/fatigue, and region loading all in a single while loop. Understanding tick ordering is prerequisite for most other topics.

### 2. Actor State & Animation
- **Files**: fmain.c, ftale.h, fsubs.asm
- **Data structures**: struct shape (ftale.h:57), statelist[87] (fmain.c:150), trans_list[9] (fmain.c:139)
- **Key functions**: set_encounter (fmain.c:2736), _page_det (fsubs.asm:1449)
- **Scope**: medium
- **Dependencies**: Topic 1 (game loop)
- **Notes**: Motion states (WALKING, FIGHTING, DYING, etc.) defined in ftale.h. The statelist maps (state, facing) pairs to animation figures. Actor struct is 22 bytes. 7 sprite sequence slots (PHIL through DRAGON).

### 3. Movement & Direction System
- **Files**: fmain.c, fmain2.c, fsubs.asm
- **Data structures**: xdir/ydir tables (fsubs.asm:1276-1277), com2 (fsubs.asm:1488)
- **Key functions**: set_course (fmain2.c:100), move_figure (fmain2.c:322), _newx/_newy (fsubs.asm:1280/1298), _decode_mouse (fsubs.asm:1489), _map_adjust (fsubs.asm:1359), _wrap (fsubs.asm:1349)
- **Scope**: medium (2 files, inline assembly in set_course)
- **Dependencies**: none identified
- **Notes**: Direction encoding: 0=NW, 1=N, 2=NE, 3=E, 4=SE, 5=S, 6=SW, 7=W, 9=none. set_course is ~130 lines of inline M68K assembly implementing 7 pathfinding modes (pursue, close, close+deviate, flee, bumble, face, velocity).

### 6. Map Rendering & Scrolling
- **Files**: fsubs.asm, fmain.c

### 7. Sprite Compositing & Blitter Operations

### 9. Audio System (Music Tracker)
- **Files**: gdriver.asm, fmain.c, fmain2.c
- **Data structures**: _vblank_data (gdriver.asm:492) — music engine state block with 4 voice states
- **Key functions**: _vblank_server (gdriver.asm:57), _playscore (gdriver.asm:352), _stopscore (gdriver.asm:407), _init_music (gdriver.asm:423), _wrap_music (gdriver.asm:466), _playsample (gdriver.asm:296), _stopsample (gdriver.asm:324), _set_tempo (gdriver.asm:332), _setscore (gdriver.asm:338), setmood (fmain.c:2936), effect (fmain.c:3616), read_score (fmain2.c:760)
